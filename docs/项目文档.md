# AI智能校园助手平台 — 项目文档

---

## 第一章 项目概述

### 1.1 项目背景

随着高校信息化建设的不断推进，学生在校园生活中面临着信息获取渠道分散、查询效率低下的问题。课程信息、空教室资源、校园新闻动态等分散在多个系统中，学生需要频繁切换平台。同时，人工智能技术（特别是大语言模型）的快速发展，为校园服务智能化升级提供了新的可能。

本项目旨在构建一个集校园信息查询与AI智能服务于一体的综合平台，为学生提供一站式的校园信息服务体验。

### 1.2 项目目标

1. 整合校园新闻、课程信息、教室资源等数据，提供统一查询入口
2. 接入DeepSeek大语言模型，实现AI智能问答功能
3. 利用AI生成个性化学习计划，辅助学生学习规划
4. 区分学生端与管理员端，实现内容管理权限分级

### 1.3 技术选型

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Vue 3（Composition API） | 3.4 |

### 6.2 关键技术实现

**JWT认证流程：**
1. 用户登录 → 后端验证密码(MD5) → 生成JWT(含userId+username+role)
2. 前端存储Token到localStorage → Axios拦截器自动附加Authorization头
3. 后端JwtInterceptor解析Token → 注入userId和role到request
4. /api/admin/** 路径额外校验role=admin，否则返回403

**空教室查询算法：**
1. 接收参数：教学楼(building)、星期(weekDay)、开始节次(startSlot)、结束节次(endSlot)
2. 查询classroom_booking表中该时间段有课的教室ID列表
3. 查询classroom表中状态为available且ID不在占用列表中的教室
4. 可选按教学楼过滤

**AI学习计划生成：**
1. 用户输入学习目标 → 构造Prompt → 调用DeepSeek API
2. Prompt要求AI按结构化格式输出（标题+每日计划+时间分配）
3. 后端将生成结果保存到study_plan表 → 返回给前端展示

## 第八章 项目总结

### 8.1 项目成果

本项目成功构建了一个功能完整的AI智能校园助手平台，具体成果如下：

1. **前后端分离架构**：Vue3前端（10个页面）+ Spring Boot后端（18个接口），代码总量5000+行
2. **完整的功能覆盖**：实现校园新闻、课程查询、空教室查询、AI问答、学习计划5大核心模块
3. **角色权限体系**：基于JWT的学生/管理员双角色权限控制
4. **AI深度集成**：成功对接DeepSeek大语言模型，实现智能问答和学习计划生成
5. **现代化UI**：采用智能驾驶舱风格首页设计，Element Plus组件库统一视觉风格
6. **数据库设计**：7张数据表，含完整E-R关系和数据初始化脚本

### 8.2 技术亮点

- MyBatis-Plus分页插件实现高效分页查询
- Axios拦截器统一处理Token鉴权和错误响应
- Pinia持久化插件实现用户状态本地缓存
- Spring Boot拦截器实现声明式权限校验
- 空教室查询采用"排除法"算法（全量教室 - 已占用教室）

### 8.3 不足与改进方向

1. 教室占用数据需与实际教务系统对接（当前为静态测试数据）
2. 可增加移动端适配（响应式布局）
3. AI对话可增加流式输出（SSE），提升交互体验
4. 可引入Redis缓存热点数据，提升高并发性能
5. 可增加更多AI功能（智能排课、成绩分析等）
